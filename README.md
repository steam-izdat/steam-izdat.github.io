# STEAMиздат

**Полевой мануал для инди-разработчиков, которые издают свои игры в Steam.**
Регистрация в Steamworks, работа из РФ и других стран, банки и выплаты, маркетинг,
цены и скидки, статистика, частые ошибки, термины.

Живая база знаний русскоязычного инди-сообщества. Сайт: **https://steam-izdat.github.io**

---

## Как устроено

Это статический сайт на [MkDocs Material](https://squidfunk.github.io/mkdocs-material/).
Весь контент — обычные markdown-файлы в `docs/`. Правишь markdown → пушишь в `main` →
GitHub Actions собирает и публикует сайт автоматически.

```
docs/                 контент гайда (md-файлы = источник правды)
  index.md            обложка
  01-start/ …         разделы (каждый — папка с index.md)
  stylesheets/        арт-дирекшн (extra.css)
  assets/             лого, favicon
overrides/            кастомные шаблоны темы (шрифты)
mkdocs.yml            конфиг сайта и навигация
requirements.txt      зависимости сборки
.github/workflows/    автодеплой на GitHub Pages
```

## Локальный запуск

```bash
pip install -r requirements.txt
mkdocs serve          # http://127.0.0.1:8000 с live-reload
mkdocs build --strict # проверить сборку как в CI
```

## Как добавить или поправить раздел

1. Открой нужный `docs/**/*.md`, поправь markdown.
2. Новый раздел — создай папку `docs/NN-slug/index.md` и добавь его в `nav:` в `mkdocs.yml`.
3. Выносные блоки — `<div class="cl cl-tip|cl-note|cl-warn|cl-danger" markdown> … </div>`.
4. Штампы-статусы — `<span class="stamp stamp--ok|stamp--warn|stamp--check|stamp--date">…</span>`.
5. Запушь в `main` — деплой произойдёт сам.

## Голос гайда

Пишется от лица **редакции STEAMиздат**, не от конкретного человека. На «ты», без
канцелярита. Каждый факт про банк/санкции/налоги — с датой «Актуально на ГГГГ-ММ» и,
по возможности, со ссылкой на первоисточник. Раздел про работу из РФ — самый
чувствительный: при сомнении лучше пустой пункт, чем неверный.
